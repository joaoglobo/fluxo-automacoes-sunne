Etapa 1 — Pagamento Confirmado ou Agendado para Gerar Contratos
● Ativo
Quando é ativado
Quando o Status do Ticket muda para "Pagamento Confirmado" ou "Pagamento Agendado". Reinscrição habilitada — se o ticket sair e retornar para esses status, o fluxo executa novamente.
O que acontece
1
No negócio associado: Etapa do Negócio → "Assinatura"
2
No negócio associado: Proprietário → responsável fixo configurado
3
No ticket: Proprietário → mesmo responsável fixo
4
No ticket: Status → "Gerar Contratos"
⚠️
Atenção: O responsável atribuído está fixo no workflow. Se esse usuário for desativado, as atribuições no ticket e no negócio falham silenciosamente. O vínculo entre ticket e negócio é obrigatório — sem ele, as ações no negócio são ignoradas sem erro visível.
Propriedades: Status do Ticket Etapa do Negócio Proprietário do Ticket
n8n detecta "Gerar Contratos" → gera contratos → escreve: Contratos do Ticket Gerados = verdadeiro

Etapa 2 — Gerar Contratos para Subir Contratos - ClickSign
● Ativo
Quando é ativado
Quando o ticket está em "Gerar Contratos" E o campo Contratos do Ticket Gerados é marcado como verdadeiro. Dois gatilhos de reinscrição independentes: mudança na propriedade OU mudança no Status do Ticket.
O que acontece
1
No ticket: Status → "Subir Contratos - ClickSign"
⚠️
Risco de bypass: Se um ticket for movido manualmente de volta para "Gerar Contratos" e o campo Contratos do Ticket Gerados já estiver marcado, o workflow avança imediatamente sem que os contratos sejam regenerados.

Handoff 3
Integração Externa · Handoff 3
n8n — Monitoramento de Assinaturas no ClickSign
🔶 n8n
O que dispara o n8n
Ticket em "Aguardando Assinatura". O n8n fica em espera pelo webhook do ClickSign confirmando que os contratos foram assinados por todos os signatários de todos os negócios associados.
O que o n8n faz
1
Aguarda o webhook do ClickSign (evento de assinatura concluída)
2
Valida que todos os contratos dos negócios associados estão assinados
3
Escreve: Contratos de Negócios Associados Assinados? = verdadeiro
🚨
Risco crítico: O campo espera o valor True com T maiúsculo. Se o n8n escrever true em minúsculas, o HubSpot pode não reconhecer o match. Valide o formato antes de ir para produção.
Sinal de retorno: Contratos de Negócios Associados Assinados?
HubSpot detecta sinal → avança ticket para próximo estágio

Etapa 6 — Cadastro no OPS para Aberto (Rateio)
● Ativo
Quando é ativado
Por evento: quando Negócios do Ticket Cadastrados no OPS? muda para verdadeiro. Condição de segurança: o ticket deve estar em "Cadastro no OPS".
O que acontece
1
No ticket: Status → "Aberto (Rateio)"
2
Aguarda 1 minuto (delay de segurança contra condições de corrida)
3
No negócio associado: Etapa do Negócio → estágio final do pipeline
⚠️
Atenção: O delay de 1 minuto é intencional — garante que o ticket esteja consolidado antes de propagar a mudança para o negócio. Mantenha esse delay. O vínculo entre ticket e negócio é obrigatório para a ação 3 funcionar — sem ele, o negócio não avança e ninguém é notificado.

Troubleshooting — Problemas que Podem Travar o Fluxo
🚨
Case sensitivo em "Contratos de Negócios Associados Assinados?"
O workflow da Etapa 4 espera o valor True com T maiúsculo. Se o n8n escrever true em minúsculas, o HubSpot pode não reconhecer o match e o ticket fica parado em "Aguardando Assinatura" sem nenhum alerta. Teste manualmente qual formato a propriedade aceita antes de ir para produção.
🔕
Nenhum workflow possui alerta de erro
Se o n8n não escrever uma propriedade de sinal dentro de um prazo esperado, o ticket fica parado no estágio atual sem que ninguém seja avisado. Recomenda-se criar um workflow de monitoramento separado que verifique tickets parados há mais de X horas em cada estágio intermediário.
👤
Proprietário fixo na Etapa 1
O responsável atribuído está hardcoded no workflow. Se esse usuário for desativado no HubSpot, as atribuições no ticket e no negócio param de funcionar silenciosamente, sem fallback nem notificação configurada.
🔗
Ausência de vínculo entre ticket e negócio é silenciosa
Nas Etapas 1 e 6, as ações que atualizam a Etapa do Negócio dependem de o ticket ter um negócio associado. Se não houver esse vínculo, as ações são ignoradas sem erro visível — o negócio simplesmente não avança.
📝
Template do Nome do Ticket depende de 3 campos
Se Tipo de Rateio, Gerador - Alocação e Rateio ou ID Usina - Alocação e Rateio estiverem vazios quando o n8n enviar o Link do Drive, o nome do ticket ficará incompleto. O n8n deve garantir o preenchimento dos três campos antes de disparar o sinal de retorno.
⚡
Risco de bypass na Etapa 2
Se alguém mover manualmente um ticket de volta para "Gerar Contratos" e o campo Contratos do Ticket Gerados já estiver marcado como verdadeiro de uma execução anterior, o Workflow 2 avança imediatamente o ticket sem que os contratos sejam regenerados.
🔗
O n8n nunca é chamado diretamente pelo HubSpot
Nenhum dos 6 workflows possui ação HTTP ou webhook explícito. É o n8n que monitora o HubSpot. Se o monitoramento do n8n ficar offline por qualquer motivo, o fluxo para no estágio atual sem que o HubSpot perceba — o ticket simplesmente não avança.
⏱️
O delay de 1 minuto na Etapa 6 é intencional
O intervalo entre mover o ticket para "Aberto (Rateio)" e avançar a Etapa do Negócio existe para evitar condições de corrida entre os sistemas. Não remova esse delay sem entender o impacto no pipeline de vendas.
