HubSpot
Workflow ID · 1697798163
Etapa 1 — Pagamento Confirmado ou Agendado para Gerar Contratos
● Ativo
Quando é ativado
Quando o Status do Ticket muda para "Pagamento Confirmado" ou "Pagamento Agendado". Reinscrição habilitada — se o ticket sair e voltar para esses status, o fluxo executa novamente.
O que acontece
→
No negócio associado: Etapa do Negócio → "Assinatura"
👤
No negócio associado: Proprietário → responsável fixo configurado
👤
No ticket: Proprietário → mesmo responsável fixo
→
No ticket: Status → "Gerar Contratos"
⚠️
Atenção: O responsável atribuído está fixo no workflow. Se esse usuário for desativado no HubSpot, as atribuições no ticket e no negócio falham silenciosamente. O vínculo entre ticket e negócio é obrigatório — sem ele, as ações no negócio são ignoradas sem erro visível.
Propriedades envolvidas: Status do Ticket Etapa do Negócio Proprietário do Ticket

Handoff 1 · Integração Externa
n8n — Geração dos Contratos
🔶 n8n
O que dispara o n8n
O n8n monitora o HubSpot via webhook. Quando detecta um ticket com Status = "Gerar Contratos", inicia a execução.
O que o n8n faz
📄
Busca os dados do ticket e do negócio associado via API do HubSpot
⚙️
Gera os documentos de contrato com as informações coletadas
✍️
Escreve de volta no HubSpot: Contratos do Ticket Gerados = verdadeiro
Sinal de retorno: Contratos do Ticket Gerados
2
HubSpot
Workflow ID · 1754291845
Etapa 2 — Gerar Contratos para Subir Contratos - ClickSign
● Ativo
Quando é ativado
Quando o ticket está em "Gerar Contratos" E o campo Contratos do Ticket Gerados é marcado como verdadeiro pelo n8n. Tem dois gatilhos de reinscrição independentes: mudança nessa propriedade ou mudança no Status do Ticket.
O que acontece
→
No ticket: Status → "Subir Contratos - ClickSign"
⚠️
Risco de bypass: Se alguém mover manualmente um ticket de volta para "Gerar Contratos" e o campo Contratos do Ticket Gerados já estiver marcado, o workflow avança imediatamente sem que os contratos sejam gerados novamente.
Propriedades envolvidas: Contratos do Ticket Gerados Status do Ticket

Etapa 3 — Subir Contratos - ClickSign para Aguardando Assinatura
● Ativo
Quando é ativado
Por evento: quando Contratos do Ticket no ClickSign muda para verdadeiro. Condição de segurança: o ticket deve estar em "Subir Contratos - ClickSign" — caso contrário, o workflow não executa mesmo que a propriedade mude.
O que acontece
→
No ticket: Status → "Aguardando Assinatura"
Propriedades envolvidas: Contratos do Ticket no ClickSign Status do Ticket
n8n
n8n
Handoff 3 · Integração Externa
n8n — Monitoramento de Assinaturas no ClickSign
🔶 n8n
O que dispara o n8n
Ticket em "Aguardando Assinatura". O n8n fica aguardando o webhook do ClickSign confirmar que os contratos foram assinados por todos os signatários de todos os negócios associados.
O que o n8n faz
👂
Aguarda o webhook do ClickSign (evento de assinatura concluída)
✅
Valida que todos os contratos dos negócios associados estão assinados
✍️
Escreve no HubSpot: Contratos de Negócios Associados Assinados? = verdadeiro
🚨
Risco crítico: O campo espera o valor True com T maiúsculo. Se o n8n escrever true em minúsculas, o HubSpot pode não reconhecer o match. Valide o formato antes de ir para produção.
Sinal de retorno: Contratos de Negócios Associados Assinados?

Etapa 5 — Assinados - Baixando Documentos para Cadastro no OPS
● Ativo
Quando é ativado
Por evento: quando Link - Anexos do Drive recebe qualquer valor preenchido. Condição de segurança: ticket deve estar em "Assinados - Baixando Documentos".
O que acontece (ordem real de execução)
①
Status do Ticket → "Cadastro no OPS"
②
Problemas nos Termos e Documentos? → falso (limpa pendências)
③
Nome do Ticket → [Tipo de Rateio] – [Gerador - Alocação e Rateio] – [ID Usina - Alocação e Rateio]
⚠️
Atenção: Se qualquer um dos três campos do template de nome estiver vazio, o Nome do Ticket ficará incompleto. O n8n deve garantir o preenchimento dos três antes de enviar o Link do Drive.
